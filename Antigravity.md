# 🧾 **교사용 출결/서류 자동 생성 서비스 기획서 (개발용/설명용)**

## 📌 **프로젝트 개요**

**교사용 출결 관련 서류(결석계, 조퇴계, 지각 사유서 등)를 자동 생성해주는 웹 서비스**이다.
교사가 수작업으로 문서를 작성하던 과정을 자동화하여,
**여러 학생의 서류를 한 번에 생성하고, 교사의 입력을 최소화하는 것**이 목표다.

가장 중요한 요구사항은:

> **학생 개인정보는 절대 서버를 거치지 않는다.**
> → 모든 학생 관련 데이터는 브라우저(로컬)에만 존재한다.

---

# 🎯 **핵심 목표**

1. **교사의 입력 최소화**
2. **한 번에 다수 학생 서류 생성**
3. **교사 환경에 맞춘 실제 HWP 출력 가능**
4. **학생 개인정보 100% 로컬 처리**
5. **템플릿 기반 자동 문서 생성**

---

# 🏛 **전체 아키텍처 요약**

```
 ┌──────────────────────────────────┐
 │            Backend API           │
 │  (Spring Boot / DB)              │
 │----------------------------------│
 │ • 학교/양식 템플릿 관리           │
 │ • 템플릿 파일(HWP/HWXP/DOCX) 제공 │
 │ • 서버는 학생 정보를 알지 못함    │
 └──────────────────────────────────┘

               ▼ 템플릿 다운로드

 ┌──────────────────────────────────┐

 │----------------------------------│

 │ • 출결 입력 UI (체크박스 + 기본값) │
 │ • 템플릿 자리표시자 채움          │
 │ • 문서 생성(DOCX/HWXP/HWP)        │
 │ • PDF/출력은 교사 PC에서            │
 └──────────────────────────────────┘
```

# 🔒 **개인정보 처리 원칙**

| 항목                 | 서버       | 프론트    |
| ------------------ | -------- | ------ |
| 학생 이름              | ❌ 저장 안 함 | ✔ 저장   |
| 학번, 생년월일           | ❌        | ✔      |
| 출결 유형/사유           | ❌        | ✔      |
| 서류 완성본             | ❌        | ✔ 다운로드 |
| 템플릿(HWP/HWXP/DOCX) | ✔ 저장     | ✔ 다운로드 |

> 서버가 받는 body에는 절대 학생 관련 필드를 포함시키지 않는다.

# 📝 **주요 기능(사용자 관점)**

## 1. **초기 세팅(회원가입 없음)**

* 학교명, 학년/반, 교사명만 입력
* 학생 명단 엑셀 업로드
* 모든 데이터는 브라우저 IndexedDB/localStorage 저장
* 백업/복원(JSON export/import)

---

## 2. **출결 서류 생성 플로우**

1. 날짜, 출결 유형, 사유 preset 선택
2. 학생 리스트에서 체크박스로 대상 선택
3. 필요 학생만 개별 수정
4. "서류 생성" → 다수 문서 자동 생성
5. DOCX/HWXP/HWP로 다운로드
6. 교사 PC의 ‘한글’에서 열어 출력/PDF 저장

# 📄 **문서 템플릿 시스템**

## 템플릿 파일 포맷

* **.hwp / .hwpx / .docx 중 선택 가능**
* 내부에는 자리표시자 포함:

```
{{schoolName}}
{{grade}}
{{classNo}}
{{teacherName}}
{{studentName}}
{{studentNo}}
{{date}}
{{reason}}
```

---

## 템플릿 처리 전략

### 🟦 백엔드(서버)에서 하는 일

* 템플릿(HWP/HWXP/DOCX) 원본 저장
* HWP → HWXP 변환 가능(하이브리드)
* HWXP → DOCX 변환도 가능(필요 시)
* 교사·학교 정보 일부만 미리 채워진 기본 템플릿 제공
* **서버는 학생 이름을 절대 채워넣지 않는다**

---

### 🟩 프론트에서 하는 일

* React에서 템플릿 파일 다운로드

* **브라우저 안에서 자리표시자 치환**

  * docxtemplater (DOCX)
  * XML 조작(HWXP)
  * hwplib(HWP, 가능 범위 내에서)

* 글씨 크기/스타일은 **템플릿에 설정된 스타일 그대로 따라감**
  → 치환된 텍스트는 해당 문단/셀의 스타일을 자동 상속함

* 다수 학생에 대해 템플릿 복제하여 ZIP 생성

---

# 🔍 **HWP / HWXP / DOCX 기술 전략**

## ✔ HWP

* hwplib 사용하여 텍스트 치환 가능
* 글씨 크기, 줄 간격 등 기존 스타일 유지 가능
  → 치환된 부분은 동일 문단 스타일 따라감
* HWP → PDF 변환은 불완전 (서버 자동화 어려움)

## ✔ HWXP

* 사실상 XML 기반(HWPX = HWP 오픈 표준)
* 자리표시자 치환 쉬움
* DOCX처럼 브라우저에서 조작하기도 편함

## ✔ DOCX

* 처리 난이도가 가장 낮음
* docxtemplater로 브라우저 안에서 완성본 생성 가능
* 학교에서 DOCX를 한글로 열어 출력 가능

> **최종 추천 구조**:
> **템플릿 관리는 HWP/HWXP 모두 OK → 문서 생성은 DOCX 기반으로 진행**

# ⚙ **백엔드 구조**

### API 예시

| Method | Endpoint                      | 설명                         |
| ------ | ----------------------------- | -------------------------- |
| GET    | `/templates`                  | 템플릿 목록 조회                  |
| GET    | `/templates/{id}`             | 템플릿 파일 다운로드                |
| POST   | `/templates/{id}/fill-global` | 학교/교사 정보만 채워진 템플릿 반환(학생 X) |
| POST   | `/templates/upload`           | 템플릿(HWP/HWXP/DOCX) 업로드     |

### DB 내용

* Template(id, name, type, filePath)
* SchoolTemplate(schoolId, templateId)
* TemplateVariables([{key:"studentName", required:true}, …])
