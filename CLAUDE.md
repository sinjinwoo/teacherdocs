# 🧾 **교사용 출결/서류 자동 생성 서비스 기획서 (개발용/설명용)**

## 📌 **프로젝트 개요**

**교사용 출결 관련 서류(결석계, 조퇴계, 지각 사유서 등)를 자동 생성해주는 웹 서비스**이다.
교사가 수작업으로 문서를 작성하던 과정을 자동화하여,
**여러 학생의 서류를 한 번에 생성하고, 교사의 입력을 최소화하는 것**이 목표다.

가장 중요한 요구사항은:

> **학생 개인정보는 절대 서버를 거치지 않는다.**
> → 모든 학생 관련 데이터는 브라우저(로컬)에만 존재한다.

---

# 🎯 **핵심 목표**

1. **교사의 입력 최소화**
2. **한 번에 다수 학생 서류 생성**
3. **교사 환경에 맞춘 실제 HWP 출력 가능**
4. **학생 개인정보 100% 로컬 처리**
5. **템플릿 기반 자동 문서 생성**

---

# 🏛 **전체 아키텍처 요약**

```

 │ • 서버는 학생 정보를 알지 못함    │
 └──────────────────────────────────┘

 ┌──────────────────────────────────┐

 │ • 템플릿 자리표시자 채움          │
 │ • 문서 생성(DOCX/HWXP/HWP)        │
 │ • PDF/출력은 교사 PC에서            │
```

# 📝 **주요 기능(사용자 관점)**

## 1. **초기 세팅(회원가입 없음)**

* 학교명, 학년/반, 교사명만 입력
* 학생 명단 엑셀 업로드
* 모든 데이터는 브라우저 IndexedDB/localStorage 저장
* 백업/복원(JSON export/import)

---

## 2. **출결 서류 생성 플로우**

1. 날짜, 출결 유형, 사유 preset 선택
2. 학생 리스트에서 체크박스로 대상 선택
3. 필요 학생만 개별 수정
4. "서류 생성" → 다수 문서 자동 생성
5. DOCX/HWXP/HWP로 다운로드
6. 교사 PC의 ‘한글’에서 열어 출력/PDF 저장

# 📄 **문서 템플릿 시스템**

## 템플릿 파일 포맷

* **DOCX 파일만 사용** (레이아웃 안정성을 위해)
* 내부에는 **짧은 영문 자리표시자** 포함

---

## 📌 **표준 변수 매핑표**

템플릿 작성 시 아래 **짧은 변수명**을 사용하여 레이아웃 깨짐을 방지합니다.

### **학생 정보**
| 변수 | 의미 | 예시 | 비고 |
|------|------|------|------|
| `{{g}}` | 학년 | 3 | grade |
| `{{c}}` | 반 | 2 | class |
| `{{sn}}` | 학생번호 | 15 | student number |
| `{{nm}}` | 학생이름 | 홍길동 | name |

### **날짜 정보 (결석 기간)**
| 변수 | 의미 | 예시 | 비고 |
|------|------|------|------|
| `{{sy}}` | 시작 년도 | 2025 | start year |
| `{{sm}}` | 시작 월 | 11 | start month |
| `{{sd}}` | 시작 일 | 28 | start day |
| `{{ey}}` | 종료 년도 | 2025 | end year |
| `{{em}}` | 종료 월 | 11 | end month |
| `{{ed}}` | 종료 일 | 30 | end day |

### **날짜 정보 (작성일)**
| 변수 | 의미 | 예시 | 비고 |
|------|------|------|------|
| `{{ty}}` | 오늘 년도 | 2025 | today year |
| `{{tm}}` | 오늘 월 | 11 | today month |
| `{{td}}` | 오늘 일 | 28 | today day |

### **기타 정보**
| 변수 | 의미 | 예시 | 비고 |
|------|------|------|------|
| `{{r}}` | 결석 사유 | 병원 진료 | reason |
| `{{d}}` | 첨부 서류명 | 진단서 1부 | document |
| `{{tn}}` | 담임교사 이름 | 김선생 | teacher name |
| `{{sc}}` | 학교명 | ○○고등학교 | school |

### **템플릿 작성 예시**

```
제 {{g}}학년 {{c}}반 {{sn}}번 이름 {{nm}}

결석일: {{sy}}년 {{sm}}월 {{sd}}일부터 (   )일간
       {{ey}}년 {{em}}월 {{ed}}일까지

위 학생은 (① {{r}})(으)로 인하여
상기 일자에 결석하였음을 확인합니다.
※ 첨부 서류: {{d}}

{{ty}}년 {{tm}}월 {{td}}일
담임 {{tn}}인
```

**장점:**
- 짧은 변수명으로 레이아웃 안정성 확보
- 한글 변수명 문제 없음 (영문/숫자만 사용)
- 자동 추출 가능 (정규식 호환)

## 템플릿 처리 전략

### 🟦 백엔드(서버)에서 하는 일

* 템플릿(HWP/HWXP/DOCX) 원본 저장
* HWP → HWXP 변환 가능(하이브리드)
* HWXP → DOCX 변환도 가능(필요 시)
* 교사·학교 정보 일부만 미리 채워진 기본 템플릿 제공
* **서버는 학생 이름을 절대 채워넣지 않는다**

---

### 🟩 프론트에서 하는 일

* React에서 템플릿 파일 다운로드

* **브라우저 안에서 자리표시자 치환**

  * docxtemplater (DOCX)
  * XML 조작(HWXP)
  * hwplib(HWP, 가능 범위 내에서)

* 글씨 크기/스타일은 **템플릿에 설정된 스타일 그대로 따라감**
  → 치환된 텍스트는 해당 문단/셀의 스타일을 자동 상속함

* 다수 학생에 대해 템플릿 복제하여 ZIP 생성

---

# 🔍 **HWP / HWXP / DOCX 기술 전략**

## ✔ HWP

* hwplib 사용하여 텍스트 치환 가능
* 글씨 크기, 줄 간격 등 기존 스타일 유지 가능
  → 치환된 부분은 동일 문단 스타일 따라감
* HWP → PDF 변환은 불완전 (서버 자동화 어려움)

## ✔ HWXP

* 사실상 XML 기반(HWPX = HWP 오픈 표준)
* 자리표시자 치환 쉬움
* DOCX처럼 브라우저에서 조작하기도 편함

## ✔ DOCX

* 처리 난이도가 가장 낮음
* docxtemplater로 브라우저 안에서 완성본 생성 가능
* 학교에서 DOCX를 한글로 열어 출력 가능

> **최종 추천 구조**:
> **템플릿 관리는 HWP/HWXP 모두 OK → 문서 생성은 DOCX 기반으로 진행**

# ⚙ **백엔드 구조**

## 🛠 **개발 환경**

### 기술 스택
- **언어/프레임워크**: Java 17, Spring Boot 3.5.8
- **데이터베이스**: PostgreSQL 16 (JSONB 지원)
- **빌드 도구**: Gradle 8.5
- **컨테이너**: Docker Compose

### Docker Compose 구성
```yaml
services:
  postgres:  # PostgreSQL 16
  app:       # Spring Boot Application
```

### 파일 저장 전략
- **DOCX 파일만 사용** (업로드/다운로드)
- **PostgreSQL BYTEA**로 바이너리 저장
- 파일 시스템 사용 안 함 (DB에 통합 저장)

---

## 📊 **실제 DB 스키마**

### School (학교)
```java
- id: Long (PK, Auto Increment)
- name: String (학교명)
```

### Docs (문서 템플릿)
```java
- id: Long (PK, Auto Increment)
- name: String (문서명)
- school_id: Long (FK → School)
- file_data: BYTEA (DOCX 바이너리)
- variables: JSONB ([{key:"studentName", required:true}, ...])
- created_at: Timestamp (자동 생성)
```

**특징:**
- `file_data`: PostgreSQL BYTEA로 DOCX 저장
- `variables`: JSONB로 자리표시자 정보 저장
- 파일 타입 필드 없음 (DOCX만 사용)

### API 엔드포인트

#### School API
| Method | Endpoint           | 설명         |
| ------ | ------------------ | ---------- |
| POST   | `/api/schools`     | 학교 생성      |
| GET    | `/api/schools`     | 학교 목록 조회   |
| GET    | `/api/schools/{id}`| 학교 상세 조회   |
| DELETE | `/api/schools/{id}`| 학교 삭제      |

#### Docs API
| Method | Endpoint                   | 설명                      |
| ------ | -------------------------- | ----------------------- |
| POST   | `/api/docs`                | 문서 업로드 (multipart/form-data) |
| GET    | `/api/docs`                | 전체 문서 목록              |
| GET    | `/api/docs/school/{schoolId}` | 특정 학교 문서 목록          |
| GET    | `/api/docs/{id}`           | 문서 메타데이터 조회           |
| GET    | `/api/docs/{id}/download`  | DOCX 파일 다운로드           |
| DELETE | `/api/docs/{id}`           | 문서 삭제                  |

**업로드 요청 예시:**
```javascript
FormData:
  - data: JSON { name, schoolId, variables }
  - file: DOCX 파일
```
